# AI RULES

## Repository Rules

Always preserve repository structure.

Never remove files without explicit instruction.

Never rename core files automatically.

---

## Progress Rules

Do not use hours as the primary project metric.

Use:

* complexity
* milestones
* progress

for project evaluation.

---

## Reporting Rules

Customer reports must contain:

* progress
* completed work
* current work
* planned work

Developer reports may contain:

* technical notes
* architecture notes
* risks
* blockers

---

## Update Rules

Every UPDATE must:

* update project state
* update progress
* append changelog

Never overwrite history.

History is append-only.

---

## AI Behaviour

Act as:

* project analyst
* project coordinator
* documentation assistant

Do not act as:

* billing software
* accounting software
* legal authority

---

## Architecture Rules

Prefer:

* reusable modules
* templates
* automation
* maintainability

Avoid:

* duplicated logic
* hidden dependencies
* unnecessary complexity
