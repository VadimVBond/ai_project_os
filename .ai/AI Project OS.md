# AI Project OS Repository Generator
ai_project_os
Создай production-ready репозиторий-шаблон под названием "AI Project OS".

Цель системы:
Создать универсальную систему управления проектами, которая может использоваться для любых проектов:

* Telegram Bot
* WebApp
* Next.js
* Python
* MkDocs
* API
* Automation
* SaaS
* Mobile
* Documentation

Система должна быть независима от технологий.

## Основные требования

Создай структуру репозитория:

/dashboard
/data
/templates
/prompts
/docs
/scripts

### Dashboard

Создай современный dashboard на:

* HTML
* CSS
* Vanilla JavaScript

Без backend.

Dashboard должен читать данные из JSON.

Поддержка:

* общий прогресс проекта
* список модулей
* done
* in_progress
* todo
* timeline
* changelog
* клиентский режим
* разработческий режим
* светлая тема
* тёмная тема
* мультиязычность (en/ru/uk)

### Data Layer

Создай универсальную схему:

project.json
portfolio.json
clients.json

Используй JSON Schema.

### AI Layer

Создай папку prompts.

Добавь:

* system_prompt.md
* init_prompt.md
* update_prompt.md
* status_prompt.md
* reevaluate_prompt.md

Все промпты должны соответствовать Universal AI Project Protocol.

### Templates

Создай шаблоны проектов:

* website
* landing_page
* telegram_bot
* webapp
* mkdocs
* nextjs
* python_backend
* saas

Каждый шаблон должен содержать:

* типовые модули
* complexity
* milestones
* client views

### Reporting

Создай систему генерации:

* client_report.md
* developer_report.md

на основе project.json

### Documentation

Создай подробную документацию:

* установка
* настройка
* workflow
* примеры использования
* интеграция с AI
* интеграция с GitHub
* интеграция с Telegram

### Scripts

Создай:

init_project
update_project
generate_report
export_client_view

для Python.

### Важные правила

Не использовать часы как основную метрику.

Основная модель:

* modules
* complexity
* progress
* milestones

Все файлы должны быть полностью реализованы.

Покажи итоговую структуру репозитория.

Сгенерируй содержимое всех ключевых файлов.

Следуй лучшим практикам архитектуры и поддерживаемости.
